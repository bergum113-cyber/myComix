## 📋 개요

**myComix**는 압축 파일(ZIP, CBZ, RAR, 7Z), 이미지 폴더, PDF, EPUB, TXT, 동영상 등 다양한 형식을 지원하는 개인 미디어 서버입니다. 자체 호스팅 환경에서 만화책, 전자책, 동영상, 문서 등을 통합 관리하고 웹 브라우저로 편리하게 열람할 수 있습니다.

> **이 저장소는 [imueRoid/myComix](https://github.com/imueRoid/myComix) (v0.488)에서 포크하여 대규모로 확장·재구성한 버전입니다.** 원본의 만화 뷰어 기능을 기반으로 보안, 성능, 멀티미디어 지원, 관리 기능 등을 전면 강화했습니다.

## 🚀 설치

1. 웹 서버 루트 또는 하위 디렉토리에 파일 업로드
2. `src/` 폴더에 쓰기 권한 부여 (`chmod 755 src/`)
3. `cache/` 폴더에 쓰기 권한 부여 (`chmod 755 cache/`)
4. `config.php`에서 `base_dirs` 경로 설정
5. 웹 브라우저로 `index.php` 접속 → 초기 관리자 계정 생성
6. 관리자 패널에서 권한, 보안, 브랜딩 설정

> 원본은 `$base_dir`(단수)였지만, 포크는 `$base_dirs`(복수)로 다중 폴더 지원합니다.

## 📝 변경 이력

### v2.4.2 (2026-06-12 ~ 06-14)

**버그 수정/개선 — toon(웹툰) 모드 위치 저장·복원 (점진 로딩 환경 대응)**
- toon 뷰어는 이미지를 **점진적으로 로딩**함(진입 시 일부만 DOM에 렌더, 스크롤하면 `appendImages()`로 더 추가, 데이터는 0번부터 순차 로드). 기존 위치 저장 로직은 "모든 이미지가 처음부터 존재한다"고 가정해 고정 범위(`image_counter`)를 돌며 `$("#image"+j).position().top`을 비교했는데, ① 갱신 조건(`.navbar` 존재 시에만)으로 전역 `scroll_top`이 0으로 남으면 항상 첫 페이지(image0)가 저장되고, ② 아직 로드/추가되지 않은 깊은 페이지 이미지에서 위치 계산이 어긋났음. 또한 진입 시 깊은 페이지(`#imageN`)는 요소 자체가 없어 복원도 되지 않았음.
- **저장**(`autosave()` 자동, `save_bookmark()` 수동): 고정 범위 루프 대신 **현재 DOM에 존재하는 이미지(`#lightgallery img[id^='image']`)만 스캔**해 `getBoundingClientRect()`로 뷰포트 상단에 걸친 마지막 이미지를 현재 위치로 기록하도록 변경. 미로딩 요소 접근 크래시 없이, 스크롤해 지나온(=로드된) 페이지를 정확히 저장.
- **복원**: 진입 시 URL의 `#imageN`을 읽어, 대상 요소가 생길 때까지 `appendImages()`로 DOM을 확장한 뒤 해당 이미지로 `scrollIntoView`. 상단 이미지 로드로 높이가 변하는 동안 잠시 재정렬하고, **대상 로드 완료/사용자 직접 스크롤/전체화면 갤러리 열림/최대 30초 타임아웃** 시 종료. 실패해도 최상단에 머무를 뿐이라 무해(fail-safe). book 분기·navbar 핸들러·즐겨찾기·전체화면 갤러리 동작은 변경 없음.
- (이 동작은 업스트림 원본과 동일했던 부분으로, 본 버전에서 점진 로딩에 맞게 견고화. 실제 스크롤 복원 동작은 실기기/브라우저 확인 권장.)

**버그 수정 — iOS 모바일, 탭 폐기 후 복귀 시 하얀 화면 + 뒤로가기를 두 번 눌러야 목록으로 가던 문제**
- **toon(웹툰) 뷰어에서 탭을 한참 백그라운드에 둬 iOS가 탭을 폐기(discard)한 뒤 복귀했을 때, 화면이 하얗게 뜨고 뒤로가기 한 번으로 목록에 못 가고 두 번째 눌러야 목록으로 가던 문제 수정.** 원인은 `viewer.php`가 페이지를 떠날 때/백그라운드 전환 시 `history.replaceState`로 현재 URL을 `mode=toon#bookmark`로 재작성한 것. 처음엔 이 `replaceState`가 `beforeunload`에 있었고(폐기 시엔 잘 실행되지 않아 원래 화면이 보였음), 즉시-복귀 시의 뒤로가기 방해를 없애려 **`visibilitychange`(백그라운드 전환 시)**로 옮겼더니, 이번엔 백그라운드로 갈 때마다 URL이 바뀌어 **iOS가 탭을 폐기하고 복귀할 때 그 바뀐 URL을 다시 로드하면서 하얀 화면 + 뒤로가기 2번**이라는 부작용이 생김. 위치 저장/복원은 이미 **자동저장(`sendBeacon`, 서버 저장)과 위치 복원(`restoreToonPosition`)**이 담당하므로 이 URL 재작성은 불필요한 중복이라 판단, **`visibilitychange`의 `replaceState` 블록을 제거**함. 이로써 백그라운드 시 URL을 바꾸지 않아 폐기 복귀 시 **원래 화면이 그대로 로드되고**(v2.4와 동일), 히스토리도 건드리지 않아 **뒤로가기 1번**으로 목록 이동. (전체화면 갤러리 닫기 시의 `location.replace`는 별개 기능으로 유지. iOS 실기기 재현 검증은 배포 후 확인 필요.)

**버그 수정 — 자동저장/북마크 최대개수 밀어내기 off-by-one (기존 항목 재저장 시 다른 항목이 사라지던 문제)**
- `bookmark.php`의 자동저장·수동북마크 저장에서, 목록이 최대치(예: 10개)일 때 **현재 파일이 이미 목록에 있는지 확인하지 않고** 무조건 가장 오래된 항목을 먼저 제거(`array_shift`)한 뒤 현재 항목을 갱신했음. 그 결과 10개가 찬 상태에서 **이미 본 만화를 다시 열면(자동저장) / 이미 북마크한 위치를 다시 저장하면** 엉뚱한 오래된 항목 하나가 사라져 9개로 줄고, 계속 기존 항목만 다루면 9개에 머물렀음. 밀어내기 조건에 `!isset($arr[$getfile])`(현재 파일이 '새 항목'일 때만)를 추가해, **새 항목이 한도를 넘길 때만** 가장 오래된 것을 제거하고 기존 항목 재저장 시에는 개수가 유지되도록 수정(항상 최대치 유지). 자동저장·북마크 두 곳 동일 적용.

**기능 추가 — 북마크 목록(드롭다운)에 사용중/최대 개수 표시**
- 기존엔 즐겨찾기에만 `⭐ N/최대` 카운트 행이 있었음. **자동저장(`⚠ N/max_autosave`)·북마크(`🔖 N/max_bookmark`)에도 카운트 행을 추가**(즐겨찾기와 동일 형식, `index.php`에서 `get_app_settings('max_autosave'/'max_bookmark')` 읽음). 표시(서버 렌더) 추가일 뿐 저장 로직과 무관.
- 옛 버전에서 만들어진 **문자열 형식 자동저장 항목**이 있으면 행 렌더(배열 가정)가 깨질 수 있어, 로드 시 배열로 정규화(`['bookmark'=>값,'viewer'=>'toon','page_order'=>'0','bidx'=>0]`)하도록 보강(북마크 렌더는 원래 `is_array` 분기가 있어 안전).

**변경 — 북마크 목록에서 EPUB/TXT 진행 항목 전체 숨김**
- EPUB/TXT 뷰어 자체가 진행률 배지·이어보기를 제공하므로, 북마크 목록의 EPUB/TXT 진행 섹션(개수 표시 + 진행률 행) 전체를 `if(false && …)`로 비활성화(코드는 보존, 필요시 `false && ` 제거로 복구). 드롭다운이 **열리는 조건**에서도 `epub_progress/txt_progress`를 제외해, EPUB/TXT 진행만 있고 만화 북마크/자동저장/즐겨찾기가 없으면 버튼이 뜨지 않도록 정리.

**버그 수정/개선 — 수동 북마크 추가 후 목록에 즉시 반영되지 않던 문제(bfcache stale) + 새로고침 없는 갱신**
- 뷰어에서 "+"로 북마크 저장(서버엔 저장됨) 후 뒤로가기로 목록(`index.php`)에 돌아오면, **bfcache 복원**이라 옛 목록이 그대로 보였음(복원 시 즐겨찾기만 동기화하고 북마크/자동저장 목록은 갱신 안 함). `save_bookmark()`(수동)와 `autosave()`(자동) 모두 저장 시 `sessionStorage` 플래그를 남기고, `index.php`의 `pageshow`(bfcache 복원)에서 그 플래그가 있을 때만 **`refreshBookmarkTable()`로 목록을 갱신**하도록 수정(수동/자동 어느 쪽이 바뀌어도 갱신됨).
- 갱신은 **전체 새로고침 없이** 현재 페이지를 `XMLHttpRequest`로 다시 받아 `#bookmarkTable` 내용만 교체(서버 렌더링을 그대로 재사용하므로 삭제 토큰·`encode_url`·링크가 동일, JS에서 토큰/인코딩 재현 안 함). 응답 실패·구조 변동 등 어떤 예외든 `location.reload()`로 안전 폴백하여 최악의 경우도 기존(전체 새로고침) 동작과 동일. 플래그가 없으면 기존대로 즐겨찾기만 동기화(불필요한 요청 없음).
- 추가 보강: 플래그 처리를 bfcache 복원뿐 아니라 **일반 로드(`pageshow` persisted=false)에서도** 수행하고, 자동저장 저장(`$.get`)이 **비동기**라 이탈 직후엔 아직 끝나지 않을 수 있어 **약간의 지연(약 0.7초) 후 갱신**해 옛 데이터를 받아오는 race를 완화함.
- 근본 보강: 페이지 이탈/숨김 시 자동저장 전송을 **`navigator.sendBeacon`으로 변경**(viewer.php). 기존 비동기 `$.get`은 뒤로가기 등으로 페이지를 떠나면 요청이 취소되어 저장이 누락/지연될 수 있었으나, `sendBeacon`은 이탈 중에도 전송이 보장됨(같은 출처라 세션 쿠키 동반, autosave는 토큰 없이 세션만으로 허용). `sendBeacon` 미지원 또는 큐잉 실패 시에만 기존 `$.get`으로 폴백하여 무회귀. (전송은 보장되나 서버 기록 완료 시점까지 100% 보장되는 것은 아니므로, 위 0.7초 지연을 함께 유지해 보완.)

**버그 수정 — iOS 모바일 사생활 보호모드 탭 복귀 시 간헐적으로 홈으로 이동하던 문제**
- **로그인 유지 상태에서 탭 복귀 시 보던 화면 대신 홈(index.php)으로 가던 문제 수정** — 사생활 보호모드는 백그라운드 전환 시 `blank.php?return=<보던 화면 URL>`로 이동했다가, 복귀 시 `session_check.php`로 세션을 확인해 유효하면 `return` 화면으로 되돌아감. 그런데 iOS에서 탭을 빠르게 복귀할 때(또는 백그라운드 탭 폐기 후 재로딩 시) 세션 확인이 그 순간 일시적으로 `false`로 떨어지는 경쟁(race) 상황이 발생할 수 있었음. 이때 `blank.php`가 `login.php`로 보냈고, **로그인 유지(remember_token)** 쿠키가 있으면 `login.php`가 자동 로그인 후 무조건 `index.php`(홈)로 보내 보던 화면을 잃었음(일반 로그인은 토큰이 없어 로그인창에 머무르므로 이 증상이 없었고, 그래서 시간과 무관하게 무작위로 갈렸음). ① `blank.php`가 로그인으로 보낼 때 보던 화면 URL(`return`)을 함께 전달하고, ② `login.php`가 자동 로그인 성공(remember_token) 및 이미 로그인된 것으로 보이는 복귀 경로에서 홈 대신 그 `return` 화면으로 복귀하도록 수정. 외부 사이트로의 오픈 리다이렉트를 막기 위해 `return`은 **같은 호스트의 절대경로(`/`로 시작)+쿼리만** 허용하고(CRLF 제거, 백슬래시 포함·프로토콜 상대 경로·`@` 우회 차단), 없거나 외부면 기존대로 `index.php`로 폴백. 사용자가 직접 로그인 폼을 제출하는 능동 로그인 경로는 기존대로 홈 이동을 유지.

**버그 수정 — 북마크 드롭다운(자동저장·북마크·즐겨찾기) 제목이 현재 폴더/구분자에 따라 잘려 보이던 문제(예: `(319967)`만 표시) + 파일명 기반 XSS 방지**
- 제목 정리 함수 `cut_title()`은 파일 그리드용으로, 현재 폴더명(`$_GET['dir']`)·구분자(`|`,`｜`,` l `)·괄호를 잘라냄(그리드에선 적절). 그러나 북마크 드롭다운은 **여러 폴더에 걸친 항목**이 섞여 있어, 폴더명이 제목과 겹치거나(`Matsuura Madoka` 폴더의 `… by Matsuura Madoka (319967)` → `(319967)`) 구분자가 들어가면 앞부분이 통째로 잘려 식별이 어려웠음. 드롭다운은 "어느 만화인지 식별"이 목적이므로, **확장자만 제거한 실제 파일명을 그대로 보여주는 전용 함수 `mark_display_title()`(function.php)를 새로 만들어 드롭다운 6곳의 제목 출력에 사용**. 폴더/구분자/괄호에 무관하게 항상 풀네임이 표시되고, 폴더를 옮겨도 제목이 바뀌지 않음. 또한 `mark_display_title()`은 출력 시 `h()`로 HTML 이스케이프하여, 파일명에 `<script>` 등이 포함된 경우의 저장형 XSS도 차단(기존 `echo cut_title(...)`은 이스케이프가 없었음). 파일 그리드의 제목 표시(`cut_title($fileinfo)`)는 그대로 유지되어 영향 없음.

**기능 추가 — 사생활 보호 복귀 시 읽던 스크롤 위치로 복원 (toon)**
- 모바일 사생활 보호(가림막) 후 복귀하면 항상 첫 페이지로 가던 것을, **읽던 위치로 복원**하도록 함. 뷰어가 현재 toon 스크롤 위치를 `window.myComixCurrentHash()`(autosave와 동일한 상단 이미지 판별)로 제공하고, `privacy_shield.php`가 가림막(`blank.php`)으로 갈 때 **복귀 URL에만** 그 위치(`#imageN`)를 실어 보냄(뷰어 자신의 URL은 바꾸지 않아 탭 폐기 복귀 시 하얀 화면 위험 없음). 복귀 시 기존 `restoreToonPosition`이 해당 위치로 스크롤.

**기능 추가/변경 — 사생활 보호 가림막을 "탭해야 내용이 보이는 진짜 커튼"으로 강화**
- 기존에는 가림막(`blank.php`)이 복귀 즉시 자동으로 원래 화면으로 되돌아가 내용이 바로 노출됐음. 이를 **자동 복귀 제거 → 사용자가 화면을 탭해야만** 세션 확인 후 내용으로 이동하도록 변경(진짜 커튼). 또한 손가락으로 밀어 올려 **앱 자체를 백그라운드**로 보낼 때 iOS가 JS를 즉시 얼려 가림막이 안 켜지던 경우를 대비해 `privacy_shield.php`에 **`pagehide` 트리거를 추가**(기존 내부이동/500ms/jumped 가드 동일 적용). 세션 만료 시 탭하면 로그인으로 이동하는 흐름은 유지.

**기능 추가 — 사생활 보호 복귀 시 목록(index)도 읽던 스크롤 위치로 복원**
- 뷰어뿐 아니라 **목록 페이지**도 사생활 보호(가림막) 복귀 시 읽던 스크롤 위치로 돌아오도록 함. 기존 `index.php`의 `infiniteScrollState` 코드는 자리표시자 선택자(`#post-container` 등) 그대로라 동작하지 않는 미완성 코드였음(건드리지 않음). 별도로 **백그라운드 전환(hidden)/이탈(pagehide) 시 현재 스크롤을 URL별 키로 sessionStorage에 저장하고, 새 로드 시 복원**하는 독립 스크립트를 추가(썸네일 로드로 높이가 변하는 동안 잠시 재적용, 사용자가 스크롤하면 중단, 해시가 있으면 비활성). bfcache 복원은 브라우저가 스크롤을 유지하므로 영향 없음.

**버그 수정 — 사생활 보호 적용 시 뷰어에서 뒤로가기가 가림막(하얀 화면)을 거쳐 느리게 목록으로 가던 문제**
- 위 "가림막 강화"에서 추가한 `pagehide` 트리거가 **브라우저 뒤로가기/링크 이동에서도 발동**(뒤로가기는 내부 클릭으로 인식되지 않음)해, `뷰 → 가림막(blank) → 목록`으로 한 단계 더 돌며 느려지던 문제. `pagehide`에서 **화면이 `hidden`(=앱이 실제 백그라운드)일 때만** 가림막으로 이동하도록 가드를 추가(뒤로가기/링크 이동 시점은 `visible`이라 발동 안 함). → 사생활 보호 중에도 뒤로가기 시 **바로 목록**으로 이동.

**버그 수정 — 사생활 보호 복귀(위치 복원) 시 뒤쪽까지 읽은 파일에서 흰 화면이 되던 문제**
- toon 지연 로딩이 **순차식**(`image100`은 `image0~99`가 로드돼야 로드됨)이라, 위치 복원으로 **깊은 위치(`#imageN`)로 점프**하면 그 앞 이미지들이 로드되지 않아 게이트에 막혀 **대상 이미지가 영영 로드되지 못하고 흰 화면**이 되던 문제(저장 위치가 깊은 "특정 파일"에서 발생). `loadLazyImage(img, force)`에 `force` 인자를 추가해, **복원 시 대상과 주변 몇 장만 순차 게이트를 무시하고 강제 로드**하도록 함(일반 옵저버/큐 호출은 `force` 없이 그대로라 평소 순차 로딩은 변화 없음). 매 틱 재호출돼도 `loadLazyImage`가 loading/loaded면 즉시 반환하므로 무해.

**버그 수정 — 제목에 `#`가 든 파일이 사생활 보호 복귀 후 흰 화면이 되던 문제 (근본 원인)**
- 가림막(`blank.php`)에서 원래 화면으로 돌아갈 때, `URLSearchParams.get('return')`이 **이미 1회 디코딩**하는데 그 뒤에서 `decodeURIComponent`로 **한 번 더 디코딩**하고 있었음(이중 디코딩). 그 결과 파일명에 포함된 `%23`(`#`)이 생짜 `#`로 풀려 `location.replace` 시 **`file` 파라미터가 `#`에서 잘려나가** 서버가 파일을 찾지 못하고 이미지 0개 → 흰 화면이 됨(제목에 `#`가 없는 파일은 잘릴 문자가 없어 멀쩡했음 = "특정 파일에서만" 발생). 서버 디버그 로그에서 정상 파일은 `total=425`, `#` 포함 파일은 `total=0`/`idx=-1`로 확정. `blank.php`의 중복 `decodeURIComponent(returnUrl)` 한 줄을 제거하여 해결(정상 파일 동작에는 영향 없음).

**개선 — 사생활 보호 복귀 시 스크롤 위치 정밀도 향상 + 가끔 최상단으로 가던 문제 완화**
- 기존에는 복귀 위치를 **이미지 단위(`#imageN`)**로만 저장해, 긴 toon 이미지의 중간을 보고 있어도 그 이미지 **맨 위**로 정렬되어 "그 근처"로만 복귀했음. `window.myComixCurrentHash()`가 상단 이미지의 **내부 픽셀 오프셋**까지 기록(`#imageNo<offset>`)하고, `restoreToonPosition`이 해당 이미지 정렬 후 오프셋만큼 추가 스크롤하여 **정밀 복귀**하도록 함(오프셋 없는 기존 북마크 `#imageN`도 그대로 호환). 또한 복귀 직후 탭 모멘텀/잔여 터치 이벤트로 복원이 즉시 중단되어 가끔 최상단에 머물던 문제를 막기 위해, 복원 시작 후 **600ms 동안의 스크롤 이벤트는 무시**(그 후 실제 사용자 스크롤이면 중단)하도록 함.

**보안 강화 — `blank.php` 오픈 리다이렉트 방지**
- 가림막 복귀 시 `location.replace(returnUrl)`이 `return` 파라미터를 **같은 출처 검증 없이** 사용하던 부분을 보강. `login.php`와 동일 정책(같은 호스트 + `/`로 시작하는 절대경로만 허용, 백슬래시 우회 차단)으로 `returnUrl`을 검증·정규화하고, 외부/비정상 URL이면 안전한 기본 홈(`index.php`)으로 보냄. 같은 출처 복귀는 경로+쿼리+위치(`#`)를 그대로 보존하므로 정상 동작에는 영향 없음(v2.4부터 있던 잠재 위험을 닫음).

**개선 — 무언가를 탭한 직후 빠르게 앱을 백그라운드로 보낼 때 사생활 가림막이 가끔 안 켜지던 문제**
- `goBlank()`의 두 차단 가드(`isInternalNavigation`, 최근 탭 후 500ms 타이밍)가 **탭 직후 즉시 백그라운드**한 경우에도 발동해 가림막을 막던 것이 원인. 이 가드는 본래 "화면이 보이는 상태의 링크 이동" 오발동 방지용이라, **화면이 hidden으로 확정된 진짜 백그라운드**에서는 불필요함. `goBlank(forced)`를 도입해 `pagehide`가 hidden(또는 hidden 전환이 이미 시작된 `pendingHide`)인 확정 백그라운드에서는 두 가드를 우회하고 가림막을 켜도록 함(중복 방지 `jumped` 가드는 유지). 링크/뒤로가기 이동은 `visible` 상태이고 `pendingHide`도 false라 영향 없음. `pendingHide`는 `visibilitychange→hidden` 시작 시 set, visible/bfcache 복원 시 reset.

**개선 — 몇 시간 뒤 복귀 후 뒤로가기 시 목록이 흰 화면으로 깜빡이던 문제 (실제 원인은 앱의 자체 리로드)**
- index.php 상단의 `tab_init` 캐시초기화 로직이 `sessionStorage`에 플래그가 없으면 `init.php?reset_cache=1` 후 `location.reload()`를 호출하는데, iOS가 몇 시간 뒤 탭을 폐기하면 `sessionStorage`가 비워져 플래그가 사라짐 → 뒤로가기로 목록에 도착하자마자 **새 탭으로 오인해 스스로 리로드**하여 흰 화면이 깜빡였음(짧은 복귀 시엔 sessionStorage가 살아있어 리로드가 없어 정상이던 것과 일치). 내비게이션 타입이 `back_forward`(뒤로/앞으로 복귀)일 때는 캐시 초기화 fetch는 수행하되 `location.reload()`를 **생략**하도록 수정하여, 폐기 후 복귀에서도 흰 리로드가 발생하지 않게 함(새 탭/일반 로드는 기존대로 동작). `.ready` 클래스는 `<body>` 직후 인라인 스크립트가 모든 풀로드에서 동기 부여하므로 페이드인 정상.

**개선 — 앱을 밀어올려 백그라운드 후 "즉시" 재진입하면 가림막이 안 켜지던 문제 (100ms 디바운스 제거)**
- `visibilitychange`가 hidden일 때 **100ms 후 재확인**하여 `goBlank()`를 호출했는데, 그 100ms 안에 다시 앱으로 돌아오면(밀어올리자마자 재진입) 재확인 시점엔 이미 `visible`이라 가림막이 **아예 안 켜졌음**(살짝 기다리면 정상이던 것과 일치). hidden 전환 시 **즉시 `goBlank` 발동**하도록 변경하여 빠른 백그라운드/재진입에도 가림막이 확실히 켜지게 함. 링크/내부 이동은 `visibilityState`가 hidden이 되지 않아 이 경로에 도달하지 않으며, 탭 직후 빠른 백그라운드는 `pagehide`의 forced 경로가 보강하므로 정상 동작에는 영향 없음. (※ 당시 함께 쓰이던 `isInternalNavigation` 무기한 차단은 이후 아래 항목에서 제거되어, 현재는 최근 500ms 타이밍 가드로만 보류함.)

**개선 — 맨 아래(끝페이지)까지 스크롤한 상태에서 사생활 복귀 시 한참 위쪽으로 복귀되던 문제**
- 복원이 대상 이미지를 위로 정렬(+오프셋)하는데, 맨 아래는 위쪽에 뒤늦게 로드될 이미지가 가장 많아 대상이 로드된 순간 루프가 끝나면 이후 위쪽 이미지가 로드되며 대상이 아래로 밀려 결과적으로 한참 위에 남았음(얕은 위치는 위 로드량이 적어 드러나지 않던 차이). `myComixCurrentHash()`가 저장 시 문서 맨 아래면 `e` 플래그를 부가(`#imageN...e`)하고, `restoreToonPosition`이 그 경우 꼬리 이미지들을 강제 로드하며 **문서 높이가 안정될 때까지 매 틱 바닥으로 스크롤**하여 끝페이지로 정확히 복귀하도록 함. 일반(중간) 위치 복원 로직은 변경 없음(`e` 플래그 없으면 기존과 동일).

**개선 — 끝 근처(깊은 위치) 복원이 너무 느려 미완료되던 문제 (복원 한정 빠른 DOM 채우기)**
- 복원 루프가 대상 이미지가 DOM에 없을 때 `appendImages()`를 한 틱(250ms)에 한 번(5장)만 호출해, image90 같은 깊은 타겟까지 DOM이 채워지는 데 수 초가 걸렸고 그 사이 사용자가 스크롤(중단)하거나 이탈해 복원이 미완료(RESTORE_DONE 없음)되었음. `appendImages`는 동기 함수로 내부 `loading` 플래그를 호출 종료 시 즉시 해제하므로, **복원 경로에 한해** 대상이 나타날 때까지(끝 복원은 문서 끝까지) 이번 틱에서 연속 호출해 빠르게 채우도록 함(상한 가드 2000회·`currentIndex` 미증가 시 중단으로 무한루프 방지). 일반 스크롤 무한로딩 동작은 변경 없음, 이미지는 여전히 lazy라 실제 로드는 대상 주변/꼬리에 한정됨.

**개선 — 사생활 복귀 시 스크롤 위치가 매번 위로 어긋나며 "왔다갔다"하던 문제 (드리프트 방지)**
- 복원이 '대상 이미지 로드' 시점에 완료 처리되어, 그 직후 위쪽 이미지들이 lazy 로드되며 문서가 늘어나면 보던 위치가 위로 밀렸음(예: 로그에서 image42로 복원 후 수 초 만에 화면 상단이 image22로 밀려, 다음 백그라운드 시 image22가 저장→다음 복원이 22로 가는 악순환). 복원 완료 조건을 '대상 로드'가 아니라 **'재정렬에 필요한 스크롤 이동량이 ±2px 이내인 상태가 2틱 연속(=주변 레이아웃 안정)'** 으로 변경하여, 위쪽 이미지가 자리 잡을 때까지 대상을 계속 의도 위치에 고정하도록 함. 프로그램적 재정렬은 사용자 스크롤 감지(wheel/touch/key)를 트리거하지 않아 자기-중단 없음, 600ms 유예·MAX_TRIES(30초)·사용자 스크롤 중단 가드는 유지.

**버그 수정 — 사생활 간헐 미발동: 내비게이션 없는 탭(메뉴 DIV 등) 이후 백그라운드에서 가림막이 안 뜨던 문제**
- `goBlank()`가 `isInternalNavigation` 플래그를 **시간 제한 없이** 검사해 차단했는데, 이 플래그는 내비게이션을 일으키지 않는 요소(예: 메뉴/토글 DIV, role=button)를 탭해도 true로 설정되고 리셋은 화면이 visible/bfcache 복원될 때만 일어남. 그래서 그런 탭 뒤 한참 지나 앱을 백그라운드로 보내면(visibilitychange→hidden) 낡은 플래그가 가림막을 막아 사생활 보호가 간헐적으로 안 됐음(로그상 `PS:blank blocked (flag)` 직후 가림막 없이 노출 복귀). 무기한 `isInternalNavigation` 차단을 제거하고, **최근 500ms 이내 탭일 때만** 보류하는 기존 타이밍 가드로 일원화함. 링크/뒤로가기 이동은 화면이 hidden이 되지 않아 이 경로에 도달하지 않으므로 보호에 영향 없고, 확정 백그라운드의 `pagehide` forced 경로도 그대로 유지.

**버그 수정 — 세로분할(3/4) 보기 모드가 자동저장 시 0으로 떨어지던 문제**
- 뷰어는 `page_order`로 `0`(기본)·`1/2`(묶음)·`3/4`(세로분할 좌→우/우→좌)를 정식 지원(`viewer.php`의 `in_array($newpageorder,[0,1,2,3,4])`)하는데, `bookmark.php`의 `sanitize_page_order()`가 `0/1/2`만 허용하고 그 외는 `0`으로 떨어뜨려, 3/4 모드로 보다가 자동저장(또는 북마크)되면 저장값이 `0`이 되어 다음에 열 때 모드가 풀렸음. 허용 집합을 뷰어와 동일하게 `0~4`로 맞춤(자동저장·수동북마크 두 경로 모두 적용). 그 외 값(예: pdf용 `page_order=pdf`)은 기존과 동일하게 `0`으로 폴백하여 pdf 동작은 변경 없음.

**개선 — 가림막 복귀 직후 목록에서 빠른 두 번 탭 시 엉뚱한 항목으로 진입하던 문제**
- 가림막(`blank.php`)을 탭해 복귀하면, 그 탭의 고스트클릭이나 빠르게 누른 두 번째 탭이 갓 로드된 목록(`index.php`)의 항목 링크(`<a href>`/`onclick`)에 떨어져 의도치 않은 항목으로 진입했음(뷰어는 항목 링크가 적어 증상이 드묾). `blank.php`가 세션 유효로 복귀하기 직전 `sessionStorage`에 복귀 플래그를 남기고, `index.php`가 로드 시 그 플래그가 최근(1.5초 이내)이면 **450ms 동안 첫 click들을 흡수**(`preventDefault`+`stopPropagation`)하도록 함. `click`만 가로채므로 스크롤·`touch`·`privacy_shield` 로직에는 영향 없으며, 450ms 후에는 정상적으로 항목을 열 수 있음.

> 코드 변경은 `config.php`(버전)와 `blank.php`·`login.php`·`viewer.php`·`index.php`·`bookmark.php`·`function.php`·`privacy_shield.php`에 한정됩니다. v2.4.1의 변경 파일(`index.php`·`function.php`·`thumb.php`·`archive_handler.php`)은 그대로 유지되며, 그 외 파일은 v2.4 원본과 동일합니다. 실서버(iOS·실세션·PC 브라우저) 환경에서의 재현 검증은 배포 후 확인이 필요합니다.

### v2.4.1 (2026-06-07)

**버그 수정 ① 같은 이름 다른 확장자 파일 누락 (캐시·표시 2단계)**
- **압축별 캐시 키 분리 (rar/7z는 확장자 포함 경로 사용)** — 같은 이름에 확장자만 다른 압축(`만화.zip`, `만화.rar`, `만화.7z`)이 한 폴더에 있을 때, 기존에는 셋 다 확장자를 떼어낸 `만화.json` 하나를 공유하여 썸네일·페이지 수 캐시가 서로 덮어써졌음. 캐시 경로 생성을 `get_cache_json_path()` 헬퍼로 일원화하여 rar/7z는 확장자를 포함한 경로(`만화.rar.json`, `만화.7z.json`)를, zip은 기존 경로(`만화.json`)를 그대로 사용(하위 호환 유지). 캐시를 읽고 쓰는 지점(목록·썸네일·커버·뷰어·`thumb.php`) 13곳을 모두 헬퍼 경유로 통일.
- **목록 중복 제거가 확장자를 무시해 같은 이름 압축이 사라지던 문제 수정** — 위 캐시 수정 후에도 목록을 화면에 표시하기 직전의 중복 제거 단계가 파일명에서 확장자를 떼어낸 이름(`만화`)을 기준으로 비교하여 먼저 처리된 하나만 남기고 나머지를 목록에서 제외했음. 캐시 자체는 정상이므로 강제 재생성을 해도 증상이 사라지지 않던 원인. 중복 제거 기준을 확장자를 포함한 전체 파일명으로 변경하여 같은 이름의 서로 다른 압축이 각각 표시되도록 수정. 검색은 기존대로 확장자를 뗀 이름으로 매칭하므로 "만화"로 검색하면 셋 다 잡힘.

**버그 수정 ② RAR 한글·특수문자 파일명 내용 표시 불가**
- **RAR 압축 내부에 한글·특수문자 폴더명이나 한글 이미지 파일명, 알파벳 분류 폴더 등이 있으면 뷰어에서 열리지 않던 문제 수정** — ZIP·7Z는 압축 내부 파일명에 특수문자·한글이 있어도 정상적으로 읽었으나, RAR은 기존 `archive_handler.php`에서 외부 도구로 특정 파일을 추출할 때 파일명 인코딩이 일치하지 않아 추출 결과를 찾지 못하고 내용을 표시할 수 없었음. ① 추출 시 UTF-8 문자셋 옵션(`-scu`)을 적용해 목록·추출 파일명의 인코딩을 일치시키고, ② 파일명 정규화(UTF-16LE/CP949 등 보정)로 매칭을 안정화하며, ③ 이름 매칭이 실패해도 추출 폴더 재귀 검색 → 전체 추출 후 목록 순번 매칭으로 이어지는 폴백 단계를 추가(경로 구조를 유지해 하위폴더의 같은 이름 파일이 덮어써지는 충돌 방지), ④ Windows·Linux 간 명령 차이(표준오류 리다이렉션·경로 구분자·표준입력 차단)를 분기 처리하여 양쪽 환경 모두에서 동작.

> 위 항목은 v2.4 배포본 이후의 작업분을 v2.4.1로 묶은 것입니다. 같은 이름 문제는 "캐시 키 충돌"과 "목록 표시 중복 제거" 두 단계가 각각 원인이었고 둘 다 해결했습니다. 코드 변경은 `config.php`(버전)와 `index.php`·`function.php`·`viewer.php`·`thumb.php`·`archive_handler.php`에 한정되며, 그 외 파일은 v2.4 원본과 동일합니다.

### v2.4 (2026-06)

**RAR / 7Z 압축 만화 지원 (신규)**
- **RAR(.rar, .cbr) / 7Z(.7z, .cb7) 압축 파일 지원 추가** — 기존엔 ZIP(.zip, .cbz)만 만화로 인식했으나, RAR·7Z 압축도 ZIP과 동일하게 목록 표시·썸네일·페이지 뷰어·북마크까지 모두 지원. 외부 명령줄 도구(UnRAR, 7-Zip)를 통해 처리하며, 관리자 설정에서 도구 경로를 지정.
- **검증된 ZIP 처리 흐름 재사용** — 별도 코드를 새로 만들지 않고, 안정적으로 동작하던 ZIP 경로에 "압축 목록 읽기"와 "개별 이미지 추출" 두 동작만 압축 종류별로 분기. 썸네일 생성·캐시 저장·표시 등 이후 로직은 ZIP과 100% 동일하게 동작하여 안정성 확보.
- **압축 형식이 거치는 모든 경로 일괄 대응** — 메인 목록 표시, 관리자 캐시 재생성(썸네일·커버·이미지목록·동영상목록), 뷰어 페이지 추출, 캐시 사전 생성(warmup), 동영상 압축 판별, 북마크 커버 생성 등 ZIP을 처리하던 모든 지점에 RAR/7Z 대응 추가.
- **관리자 설정 안내 강화** — 압축 도구 설정 항목에 "GUI 프로그램(WinRAR.exe)이 아닌 명령줄 도구(UnRAR.exe)를 연결해야 한다"는 안내 추가. GUI 프로그램 연결 시 페이지 무한 로딩이 발생하는 문제를 사전 방지.

> ZIP 처리 코드는 원본 그대로 유지하고 RAR/7Z 분기만 추가했으므로, 기존 ZIP 만화의 동작에는 영향이 없습니다.

### v2.3 (2026-05)

**성능 최적화 (폴더 진입 속도 개선)**
- **동영상 ZIP 캐시 통과 수정** — 동영상 ZIP(`.video_files.json` 보유)이 기존엔 `.image_files.json` 부재로 캐시 통과에 실패해, 표시할 때마다 ZipArchive를 다시 열고 메타파일을 재기록했음. 이 재기록이 폴더 mtime을 갱신시켜 다음 진입 시 파일 목록 캐시가 무효화(`cache_old`)되고, 결과적으로 폴더 전체를 매번 재스캔하는 악순환이 발생. 동영상 ZIP은 `.video_files.json` 존재로 캐시를 판정하도록 보강하여 악순환 제거. (간헐적으로 상위 폴더 진입이 수 초까지 느려지던 현상 해결)
- **커버 썸네일 중복 stat 제거** — 폴더 대표 커버(`[cover].jpg`)는 폴더당 하나인데, 기존엔 표시되는 파일마다(수백~수천 회) 동일 파일을 `is_file`+`filesize`로 반복 확인했음. 폴더당 한 번만 확인하도록 변경하여 디스크 I/O 대폭 감소. (파일이 많은 폴더일수록 효과 큼)
- **NEW 딱지 시각 계산을 캐시값으로 대체** — NEW 딱지(최근 추가 표시) 판정에 파일마다 `filectime()`을 디스크에서 호출(수백 회)하던 것을, 이미 파일 목록 캐시에 저장된 `time`(filemtime) 값을 사용하도록 변경하여 디스크 접근 회피. 캐시에 값이 없는 예외 상황에서는 기존 `filectime()` 방식으로 폴백. (OS 파일 캐시가 식은 첫 접근 시 부담 감소)
- **폴더 개수 카운트 최적화** — 폴더 리스트를 그릴 때 각 폴더의 항목 수를 `scandir`(전체 배열 생성+정렬)로 세던 것을 `FilesystemIterator`(개수만 카운트)로 변경. 결과값은 동일하며, 폴더가 많은 목록일수록 디스크 I/O와 메모리 부담 감소.

> 위 최적화는 모두 동작 결과를 바꾸지 않고 디스크 접근(I/O) 횟수만 줄이는 변경입니다. 파일/폴더가 많은 환경에서 OS 파일 캐시가 식은 상태(오랜만의 첫 접근)일 때 체감 속도가 개선됩니다.
